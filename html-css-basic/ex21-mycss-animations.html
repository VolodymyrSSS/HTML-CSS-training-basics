CSS Animations

An animation lets an element gradually change from one style to another.
CSS allows animation of HTML elements without using JavaScript or Flash!
You can change as many CSS properties you want, as many times you want.
To use CSS animation, you must first specify some keyframes for the animation.
Keyframes hold what styles the element will have at certain times.

    Animation of HTML elements could have the following properties:
  1. @keyframes - required!
  2. animation-name - required!
  3. animation-duration - required!
  4. animation-delay
  5. animation-iteration-count
  6. animation-direction
  7. animation-timing-function
  8. animation-fill-mode
  9. animation

The @keyframes Rules - required ones

Some older browsers need specific prefixes (-webkit-) to understand the animation properties:

This example1 does not work in Internet Explorer 9 and earlier versions.
The animation will last for 4 seconds, and it will gradually change the background-color of the div element from 'red' to 'yellow'. When an animation is finished, it changes back to its original style - red one.

Ex1 - Square with colours changed from red to yellow
div {
width: 150px;
height: 150px;
background-color: red;
-webkit-animation-name: example1; /* Safari 4.0 - 8.0 */
-webkit-animation-duration: 4s; /* Safari 4.0 - 8.0 */n
animation-name: example1;
animation-duration: 4s;
}

/* Standard syntax */
@keyframes example1 {
from {
background-color: red;
}
to {
background-color: yellow;
}
}

To get an animation to work, you must bind the animation to an element.
This example1 is binded to the div-element by defining the property 'animation-name: example1;'
As well, in this example we have specified when the style will change by using the keywords "from" and "to".

When you specify CSS styles inside the @keyframes rule, the animation will gradually change from the current style to the new style at certain times.
If the animation-duration property is not specified, no animation will occur, because the default value is 0s (0 seconds).

Run Animation in Reverse Direction or Alternate Cycles

The animation-direction property specifies whether an animation should be played forwards, backwards or in alternate cycles.

    The animation-direction property can have the following values:
  1. normal - The animation is played as normal (forwards). This is default
  2. reverse - The animation is played in reverse direction (backwards)
  3. alternate - The animation is played forwards first, then backwards
  4. alternate-reverse - The animation is played backwards first, then forwards

The following example7 will run the animation in reverse direction (backwards):

Ex7
div {
width: 100px;
height: 100px;
background-color: red;
position: relative;
-webkit-animation-name: example7; /* Safari 4.0 - 8.0 */
-webkit-animation-duration: 4s; /* Safari 4.0 - 8.0 */
animation-name: example7;
animation-duration: 4s;
animation-direction: reverse;
}

/* Standard syntax */
@keyframes example7 {
0% {
........etc as before }

The following example8 uses the value 'alternate' to make the animation run forwards first, then backwards:

Specify the Speed Curve of the Animation

The animation-timing-function property specifies the speed curve of the animation.
Note: The animation-timing-funtion property is not supported in Internet Explorer 9 and earlier versions.

    The animation-timing-function property can have the following values:
  1. ease - Specifies an animation with a slow start, then fast, then end slowly (this is default)
  2. linear - Specifies an animation with the same speed from start to end
  3. ease-in - Specifies an animation with a slow start
  4. ease-out - Specifies an animation with a slow end
  5. ease-in-out - Specifies an animation with a slow start and end
  6. cubic-bezier(n,n,n,n) - Lets you define your own values in a cubic-bezier function

The following example9 shows some of the different speed curves that can be used:

ease-out

Specify the fill-mode For an Animation

CSS animations do not affect an element before the first keyframe is played or after the last keyframe is played. 'The animation-fill-mode property' can override this behavior.
The animation-fill-mode property specifies a style for the target element when the animation is not playing (before it starts, after it ends, or both).
Note: The animation-fill-mode property is not supported in Internet Explorer 9 and earlier versions as well.

    The animation-fill-mode property can have the following values:
  1. none - Default value. Animation will not apply any styles to the element before or after it is executing
  2. forwards - The element will retain the style values that is set by the last keyframe (depends on animation-direction and animation-iteration-count)
  3. backwards - The element will get the style values that is set by the first keyframe (depends on animation-direction), and retain this during the animation-delay period
  4. both - The animation will follow the rules for both forwards and backwards, extending the animation properties in both directions

The following example10 lets the div-element retain the style values from the last keyframe when the animation ends:

div {
width: 100px;
height: 100px;
background: red;
position: relative;
-webkit-animation-name: example10; /* Safari 4.0 - 8.0 */
-webkit-animation-duration: 3s; /* Safari 4.0 - 8.0 */
-webkit-animation-fill-mode: forwards; /* Safari 4.0 - 8.0 */
animation-name: example10;
animation-duration: 3s;
animation-fill-mode: forwards;
}

/* Standard syntax */
@keyframes example10 {
from {
top: 0px;
}
to {
top: 300px;
background-color: blue;
}
}
