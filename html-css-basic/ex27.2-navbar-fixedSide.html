Try to scroll this area, and see how the sidenav sticks to the page

Notice that side navigation div has a left margin of 31%. This is because this sidenav-div is set to 25% width. If you remove the margin, the sidenav-div will overlay/sit on top of other div with text.

Also notice that we have set overflow:auto to sidenav. This will add a scrollbar when the sidenav is too long (for example if it has over 50 links inside of it). 

              <ul class="vertical-navbar">
                <li><a class="active" href="#home">Home</a></li>
                <li><a href="#news">News</a></li>
                <li><a href="#contact">Contact</a></li>
                <li><a href="#about">About</a></li> </ul>
              </ul>

.vertical-navbar {
   list-style-type: none;
   margin: 0;
   padding: 0;
   width: 25%;
   background-color: wheat;
   position: fixed;
   height: 100%;
   overflow: auto;
}

.vertical-navbar > li a {
   display: block;
   color: #000;
   padding: 8px 16px;
   text-decoration: none;
   border-bottom: 2px solid brown;
}

.vertical-navbar > li a.active {
   background-color: #4caf50;
   color: white;
}

.vertical-navbar > li a:hover:not(.active) {
   background-color: #555;
   color: white;
}
